## 사내 개발자들이 시장을 이해하지 못하는 문제 해결
{{< badge style="info" title="포스페이스랩" >}}

{{% hint %}}
데이터와 IT 전문 인력이 부족한 상황에서, 복잡한 프랜차이즈 데이터 구조 때문에 내부에서도 시스템 이해가 혼란스러웠습니다.
저는 DB 구조를 파악하고, 이해관계자 질의와 업무 흐름 조사, 슬랙 기록 등을 통해 전체 시스템을 분석했습니다.
이후 직접 데이터 수집기를 개발하고 데이터 흐름 다이어그램을 작성해 조직 내 공통 이해를 만들었으며, 누락된 시스템 요소들이 반영되도록 주도했습니다.
전자공학·IT개발 방법론·프로젝트 관리 등 다양한 배경을 바탕으로 새로운 기술과 환경에 빠르게 적응하며 구조적 문제 해결에 기여했습니다.
{{% /hint %}}

## 고질적인 사내 문제점 해결
{{< badge style="info" title="마인드허브" >}}

{{% hint %}}
운영팀 전반의 혼란이 개인 문제가 아니라 운영 개념 자체의 부재에서 비롯된 것임을 파악했습니다.
정의되지 않은 업무 구조를 분석해 핵심 문제를 하나의 구조로 정리하고, 필요한 정보 체계와 프로세스를 재설계했습니다.
운영에 맞는 백오피스 시스템을 직접 구축하고, 정책·절차·역할을 명확히 해 팀이 독립적으로 기능하도록 만들었습니다.
결과적으로 혼란스러운 운영 업무를 단일한 운영 시스템으로 재구축하여 조직 전체의 안정성을 회복시켰습니다.
{{% /hint %}}

## 전산화인지재활 시장 전체의 문제점 파악
{{< badge style="info" title="마인드허브" >}}

{{% hint %}}
전산화 인지 재활이란 디지털기기를 통해 인지 장애를 치료하는 것을 말합니다. 인지 장애 환자는 해마다 전세계적으로 증가하고 있는 반면 전산화 인지 재활 시장은 부진을 면치 못하고 있는 상황입니다. 그러나 희한하게도 전산화 인지 재활 업계 플레이어들은 시장이 아직 성숙하지 않았을 뿐이라고 이야기하고 있습니다. 그래서 저는 다양한 업계 전문가들 인터뷰와 실제 데이터들을 기반으로 전산화 인지재활 시장의 문제점을 드러내고 해결방법을 제시했습니다.
{{% /hint %}}
