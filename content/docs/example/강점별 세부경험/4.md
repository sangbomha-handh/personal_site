# 복잡한 문제를 단일 구조로 압축하는 설계 능력
{{% hint info %}}
- 복잡한 요구사항과 제약조건 속에서도 핵심 변수를 추출해 구조화
- 제품·UX·데이터 흐름을 하나의 통합된 시스템으로 설계
{{% /hint %}}


## 사내 개발자들이 시장을 이해하지 못하는 문제 해결
{{% hint %}}

데이터와 IT 전문 인력이 부족한 상황에서, 복잡한 프랜차이즈 데이터 구조 때문에 내부에서도 시스템 이해가 혼란스러웠습니다.
저는 DB 구조를 파악하고, 이해관계자 질의와 업무 흐름 조사, 슬랙 기록 등을 통해 전체 시스템을 분석했습니다.
이후 직접 데이터 수집기를 개발하고 데이터 흐름 다이어그램을 작성해 조직 내 공통 이해를 만들었으며, 누락된 시스템 요소들이 반영되도록 주도했습니다.
전자공학·IT개발 방법론·프로젝트 관리 등 다양한 배경을 바탕으로 새로운 기술과 환경에 빠르게 적응하며 구조적 문제 해결에 기여했습니다.
{{% /hint %}}
