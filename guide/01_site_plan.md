#### 카드 1: 포스페이스랩 (2022-2025) — **메인 · 양쪽 트랙 공통**

**서사 위치**: 리서치 → 실행 전체 사이클. 대기업·스타트업 양쪽에서 가장 강력한 증거.

**헤드라인**: 고객 해지를 업세일로 — B2B SaaS 데이터 제품 3년

**핵심 수치**: 해지방어율 80% · 신규 중견 고객 3개사 · 창사 첫 업세일

**Situation**: 데이터팀 리더로 합류 당시 주요 고객사들의 해지 의사가 누적되던 상황. 제품팀은 "기능 부족" 가설, 영업팀은 "가격" 가설이었음.

**Action**:
- 자발적으로 고객 미팅 참여 → 문제를 **"ERP 기능 부족"이 아닌 "수동 반복 업무 부담"**으로 재정의
- 프랜차이즈 고객 데이터 구조 분석 → 업무 자동화 신규 제품 기획
- 기획·개발·영업 전 과정 직접 리드
- 사내 개발자 대상 시장 이해 체계 구축 (도메인 지식 전파)

**Result**:
- 해지 위기 고객 2개사 방어 (해지방어율 80%)
- 신규 중견 고객 3개사 유치
- 창사 이래 첫 업세일 달성
- 영업 프로세스 자체 재설계 (고객 성공 기반 영업으로 전환)

**태그**: `고객 문제 재정의` `신규 제품 0→1` `B2B SaaS` `직접 영업`

**⚠️ 사용자 필요 자료**:
- [ ] **계약 금액 / ARR 숫자**: 가능하면 "업세일 계약 규모 N억"이나 "ARR N% 증가" 같은 **비공개 범위 내 제시 가능한 수치**. NDA 범위 내에서 익명화된 표현(예: "업세일 금액은 기존 계약 대비 ~배") 권장
- [ ] **신규 제품 이름 / 간단한 설명**: 업무 자동화 제품의 명칭 (내부 코드명이라도). 공개 가능하다면 스크린샷 1장
- [ ] **포스페이스랩이 나온 이유**: 면접 단골 질문에 대한 사이트 답변 한 줄. Now 섹션에서 같이 다룰 수도 있음

#### 카드 2: LG전자 터치 정확도 (2013-2015) — **서브 · 대기업 트랙 강세**

**서사 위치**: "업계 공통 프레임을 리서치로 뒤집은 케이스." 대기업 UX/리서치 포지션에서 가장 잘 먹힘.

**헤드라인**: 터치 정확도 83.5% → 96.1% — 문제 정의 전환

**핵심 수치**: 정확도 83.5% → 96.1% (애플 벤치마크 97.1%에 근접)

**Situation**: 업계 공통 프레임은 "칩 스펙이 부족해서 정확도가 떨어진다"였음. 하드웨어 교체는 수년 단위 로드맵이라 단기 개선 불가능.

**Action**:
- 가설 전환: 문제를 **"칩 스펙"이 아닌 "사용자마다 다른 터치 패턴"**으로 재정의
- 사용자 터치 패턴 수집 앱 개발
- K-Means 군집 분석 기반 터치 알고리즘 설계
- 물리학 기반 시각·지각 실험 + 인지심리 실험법 적용
- 전자회로·기구·펌웨어·SW까지 프로토타입 직접 구현

**Result**:
- 터치 정확도 83.5% → 96.1%
- 양산 적용
- 그룹사 최우수 프로젝트 선정

**태그**: `리서치 기반 문제 재정의` `데이터 분석` `양산 적용` `다학제`

**⚠️ 사용자 필요 자료**:
- [ ] **공개 가능 여부 확인**: LG 내부 수치(83.5%, 96.1%)가 외부 공개 가능한지 확인. 사내 컨퍼런스·논문 발표 이력이 있으면 공개 안전. 없으면 "수치는 면접 시 공유" 표기 고려
