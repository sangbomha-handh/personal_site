## 3. 섹션별 작성 가이드

### 3-1. Header

**포함**:
- 한영 이름 병기
- 직무 타이틀 (사이트와 일치)
- 이메일, LinkedIn, 사이트 URL
- **선택**: 지역 (서울 또는 여주) — 채용 시 통근 가능성 판단용

**제외**:
- 생년월일, 나이 → 한국 관행이지만 **넣지 마라**. 40대가 이력서 상단에 나이 박으면 자기 컷 신청하는 거.
- 사진 → 선택. 사이트에 있으니 생략 가능. 넣는다면 사이트와 동일 컷 또는 조금 더 포멀한 버전
- 주소 → 도시까지만, 상세 주소 X
- 주민번호 → 절대 X

### 3-2. Summary (3줄)

**구조**: `[경력 요약] + [핵심 역량] + [가치 제안]`

**초안**:
> LG전자 엔지니어링 10년, 제품 창업 6년, B2B SaaS 3년의 경력을 가진 프로덕트 매니저. 고객 문제를 재정의하고 데이터 기반으로 제품을 기획·검증·출시해 해지방어와 신규 매출을 동시에 달성한 경험이 있음. 엔지니어·디자이너·영업과 같은 언어로 대화하며, 조직 내 부문 간 간극을 좁히는 역할에 강점.

**작성 원칙**:
- 첫 줄에 **근속 숫자** 박아넣기 (신뢰 시그널)
- 두 번째 줄에 **정량 성과 키워드**
- 세 번째 줄에 **차별화 포인트** (여기선 "부문 간 간극을 좁히는")
- 아포리즘 금지. 담백하게.
- 3줄 넘기지 마라. 읽는 사람 시간 없다.

### 3-3. Experience

**각 직장 작성 포맷**:
```
[회사명] | [포지션] | [기간]
```

**불릿 작성 룰 (STAR 압축)**:
- **Situation + Task + Action + Result**를 **한 불릿에 압축**
- 나쁜 예: "자동화 제품을 개발했다"
- 좋은 예: "**해지 의사 누적 고객사 대상으로** 요구사항을 재정의해 **업무 자동화 제품을 기획·개발**, **해지방어율 80% 및 창사 첫 업세일** 달성"
- 동사는 **과거형**, 주어는 생략, 숫자를 맨 뒤에

**회사당 불릿 수**:
- 포스페이스랩 (메인): 5-6불릿
- 마인드허브 (현재): 3-4불릿
- 로캣디자인: 3-4불릿
- LG전자: 4-5불릿 (내부 이동 포함)

**우선순위**:
- 타깃 직무 관련도 높은 것부터
- 정량 성과 있는 것부터
- 최근 것부터

**특별 주의**:

**마인드허브 (재직 중)**:
- 재직 중인 직장 이름을 이력서에 박는 건 일반적. 다만 **구직 사실이 샐 수 있음**.

- 단, 회사 내부에서 알면 곤란하면 익명화 + 지원 시 구두로 공개

**LG전자 2007 PDP 공정기술팀**:
- 4개월이라 짧음. **"LG전자 2007-2015"로 통합 표기**하고 내부 이동으로 자연 흡수

## 4. 한국 시장 관행 주의사항

### 나이
- 이력서 상단에 생년월일 박는 관행 있지만 **40대 경력직은 빼라**
- 경력 연수가 나이를 암시하게 두면 충분
- 서류 통과 후 면접 단계에서 자연스럽게 드러나는 게 낫다

### 사진
- 한국은 사진 요구하는 곳 많음
- **PDF에는 넣지 말고, 별도 프로필 사진 파일을 준비**해서 요구받으면 제공
- 사이트에는 이미 있으니 채용담당자가 궁금하면 사이트 방문 유도 가능

### 외국계 / 스타트업 구분
- **대기업·중견기업**: 전통 포맷(위 구조 그대로), 한글 우선
- **외국계**: 영문 이력서 필수, 사진 X, 나이 X, Summary 강화
- **스타트업**: 간결성·임팩트 중시, Skills·Tools 구체적으로, 사이드 프로젝트 일부 허용 (단 직무 관련만)

### 자기소개서
- 한국 대기업 + 중견은 이력서와 별도 자소서 요구
- 양식이 회사별로 다름 (보통 지원 동기 / 성장 과정 / 강점 / 지원 직무 이해)
- **이 가이드 범위 밖.** 별도 준비 필요
